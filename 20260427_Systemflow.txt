== ① User Authentication (Cognito + IDP) ==

User -> Cognito : Login request (SSO via IDP)
Cognito -> Cognito : MFA / OTP email verification
Cognito --> User : JWT Token (1 hour expiry)

note right of Cognito
  Authenticated Role:
  mode2-cognito-auth-role
  → execute-api:Invoke only
end note

== ② Request Routing via ALB ==

User -> ALB : HTTPS request + JWT Token
ALB -> ALB : Verify Cognito JWT
ALB -> ECS : Forward authenticated request

== ③ ECS / Fargate — Chatbot Processing ==

ECS -> AgentCore : Invoke AgentCore Runtime\n(bedrock-agentcore:*)
AgentCore -> Bedrock : InvokeModel\n(Claude Sonnet / Haiku)
AgentCore -> KB : Retrieve relevant documents\n(Titan Embeddings + S3)
KB --> AgentCore : Return context documents
Bedrock --> AgentCore : LLM response
AgentCore --> ECS : Agent response with action plan

== ④ OTP Two-Factor Authentication (for EC2 operations) ==

ECS -> Lambda : Request OTP generation\n(user email, operation type)
Lambda -> Lambda : Generate 6-digit OTP
Lambda -> DDB : Save OTP\n(TTL: 10 min, status: pending)

note right of Lambda
  ⚠ Remove before production:
  print(f"Generated OTP: {otp_code}")
end note

Lambda -> SES : Send OTP email\n(no-reply-security@mail.nissan.co.jp)
SES --> User : OTP email delivered

User -> ECS : Submit OTP code
ECS -> Lambda : Validate OTP
Lambda -> DDB : Verify OTP\n(check expiry + used flag)
DDB --> Lambda : Validation result
Lambda --> ECS : ✅ OTP verified — operation permitted

== ⑤ Cross-Account Access via sts:AssumeRole ==

ECS -> STS : AssumeRole request\n(Resource: mode2-ops-role)\n(ExternalId: mode2-chatbot-trusted)

note right of STS
  Supported target accounts:
  - B2C NonProd / Prod
  - Internal NonProd / Prod
  - Gamma NonProd / Prod
end note

STS -> TargetRole : Verify Trust Policy\n(Principal + ExternalId check)
TargetRole --> STS : Trust verified
STS --> ECS : Temporary credentials\n(valid 15 min – 1 hour)

== ⑥ EC2 / RDS Operation Execution ==

ECS -> EC2 : EC2/RDS operation\n(Start / Stop / Reboot / Describe)

note right of EC2
  IAM Condition:
  ec2:ResourceTag/mode2-managed = true
  → Only tagged instances can be operated
end note

EC2 --> ECS : Operation result
ECS -> SES : Send completion notification\n(to user + team)
SES --> User : ✅ Operation complete email
ECS --> ALB : Response
ALB --> User : Display result in browser

== CloudTrail Audit (all operations recorded) ==

note over ECS, EC2
  All operations (AssumeRole, EC2 Start/Stop, RDS operations)
  are recorded in AWS CloudTrail for audit purposes.
end note
